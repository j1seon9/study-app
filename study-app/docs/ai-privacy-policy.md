# AI 개인정보 및 데이터 수집 정책

## 목적

본 서비스는 학생의 자기주도학습을 지원하기 위해 AI를 활용한다.

AI는 학생을 식별하거나 개인정보를 분석하는 것이 아니라, 학습 데이터를 기반으로 맞춤형 학습 전략을 추천하는 것을 목적으로 한다.

---

# 개인정보 최소 수집 원칙

AI는 서비스 운영에 필요한 최소한의 정보만 사용한다.

학생을 직접 식별할 수 있는 개인정보는 AI 학습 및 예측에 사용하지 않는다.

# AI 개발 원칙

- 개인정보를 이용하여 학생을 식별하지 않는다.
- 학습 데이터만 분석한다.
- 학생 간 순위를 생성하지 않는다.
- 학생을 평가하거나 차별하는 용도로 사용하지 않는다.
- 추천 결과에는 항상 추천 이유를 제공한다.
- 시험 문제를 생성하지 않는다.
- EBS 문제은행 검색 조건만 추천한다.
- 개인정보는 모델 학습 데이터셋에 포함하지 않는다.
- 모델 재학습 시 개인정보를 제거(비식별화)한 학습 데이터만 사용한다.

# 구현 현황 메모 (2026-07)

- `StudentProfileForm`(인구통계 설문)은 완전히 제거했다. AI 리포트 생성은
  학생에게 아무 정보도 요청하지 않고, 이미 쌓인 공부 계획/기록/미니테스트/
  모의고사 데이터만으로 실행된다 (`POST /api/ai/reports` 요청 본문 없음).
- 현재 AI 모델은 UCI Student Performance 데이터셋 스키마를 기반으로 하고
  있어, 정책에 없는 인구통계 필드(성별/가족형태/부모 학력·직업/음주 빈도 등)를
  모델 입력으로 요구한다. 이 필드들은 실제 학생 데이터가 아니라 **모든
  사용자에게 동일하게 적용되는 고정 상수**(`ai.controller.js`의
  `NEUTRAL_PROFILE`, `aimodel/ai/dataset_builder.py`의 `PLACEHOLDER_PROFILE`)로
  채워 넣는다 — 특정 학생을 식별하거나 그 학생의 실제 정보를 담지 않으므로
  개인정보 수집에 해당하지 않는다.
- 다만 이는 임시 조치다. 정책의 "AI가 수집 가능한 데이터" 목록(학년, 학교급,
  목표 공부시간/평균점수/등급/백분위, 대단원/중단원/소단원 단위 학습 데이터)에
  맞춰 학습 특성(feature)을 새로 설계하고, UCI 데이터셋 의존도를 낮추는 작업이
  필요하다. `dataset_builder.py`가 이미 실제 웹서비스 데이터를 CSV로 뽑아내고
  있으므로, 다음 단계로 이 CSV의 컬럼 구성을 정책에 맞는 특성으로 교체하는
  리팩터링을 진행하는 것을 권장한다.
